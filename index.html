The dynamic sidebar

How it works

Sidebars are often used in blogs, forums or wikis. One of the main design-trends that never go wrong, is to have a sidebar that has consistent height same as the article and it's content. The usual fix for that is to set the height to 100vh(viewport height) or 100% on the height in css. You can position it fixed to allways stay reachable when schrolling down. However, some problems that can occur are if the sidebar-content are longer than the article it self or the article content is heigher than the viewportheight. That would cause an unnescesary schrollbar on the sidebar. Of course there is allways "overflow hidden", but it's going to hide information you'd want to be able to display in the sidebar.

The trick

There are probebly many solutions for this, but mine are to target the height on the article container with help of Javascript. To make the sidebar contain the same height on window resize the function is simply called again on window.resize.

Fixed bug issue

when loading fonts from a cdn such as google fonts, it take some time to load in, and therefore the script loads to fast. The key is to use body onload and call the function from there and load the cdn in the stylesheet. That will prevent the script to load in before the font has been loaded in to the document.
