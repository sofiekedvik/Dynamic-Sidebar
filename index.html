The dynamic sidebar

How it works

Sidebars are often used in blogs, forums and wikis. One of the main design trends that is to have a sidebar that has consistent height same as the article and it's content. The usual fix for that is to set the height to 100vh(viewport height) or 100% on the height in css. You can position it fixed to allways stay reachable when schrolling down. However, some problems that can occur are if the sidebar-content are longer than the article it self or the article content is heigher than the viewportheight. That would cause an unnescesary schrollbar on the sidebar. Of course there is allways "overflow hidden", but it's going to hide some of the information you'd want to be able to display in the sidebar.

The trick

There are probebly many solutions to solve this issue, but in this example we are targetting the height on the article container with some help of Javascript. To make the sidebar contain the same height on diffrent sizes of the vendor window, the function is simply called again on window.resize.

Fixed bug issue

when loading fonts from a cdn such as google fonts, it adds some loading time and the Javascript file may load in faster. This causes the script getting the height of the article before the new font is loaded, and therefore can miss a few pixels depending on the height of the font, a glitch may appear at the bottom of the sidebar. To prevent this we can use the onload function on the bodytag and load the cdn through the stylesheet instead.
